# CUDA Portfolio Review Skill

Use this skill to review documentation and claims.

## Review criteria

Check that the project includes:

- clear problem statement;
- reason the workload matters;
- CPU/reference implementation;
- correctness tests;
- benchmark methodology;
- environment metadata;
- measured results;
- profiler or bottleneck analysis;
- limitations;
- next steps.

## Overclaim detection

Flag claims that:

- lack hardware context;
- lack input shape;
- compare unfair baselines;
- exclude transfer time without saying so;
- imply general superiority from one benchmark;
- hide correctness tolerances;
- obscure Codex assistance.

## Output format

```text
Portfolio readiness score: /5
Strongest evidence:
Weakest gap:
Unsupported claims:
Suggested rewrite:
Next polish task:
Interview questions this project should prepare for:
```
